🎯 FEATURES IMPLEMENTED

  ✅ USER REGISTRATION
     • POST /api/auth/register
     • Accepts: email, password, userType (doctor/patient), fullName
     • Returns: UUID, email, userType, access_token
     • Auto-creates user profiles (doctor_profiles or patient_profiles)
     • 201 Created status code

  ✅ USER LOGIN
     • POST /api/auth/login
     • Accepts: email, password
     • Returns: access_token, user object
     • Bcrypt password verification
     • 200 OK status code

  ✅ JWT AUTHENTICATION
     • HMAC SHA-256 signing
     • 1-hour token expiration
     • Stateless architecture
     • Signed with JWT_SECRET environment variable
     • Payload: user_id, user_type, email, iat, exp

  ✅ AUTH MIDDLEWARE
     • Validates JWT tokens
     • Checks expiration
     • Protects sensitive routes
     • Attaches user info to requests

  ✅ PROTECTED ROUTES
     • GET /api/me (requires valid JWT)
     • Returns: id, email, user_type
     • 401 Unauthorized if no/invalid token

  ✅ SECURITY
     • Bcrypt hashing (cost factor 12)
     • Prepared SQL statements
     • Input validation (server-side)
     • Error message sanitization
     • CORS headers configured
     • No SQL injection vulnerability
     • No password exposure

  ✅ DATABASE
     • Automatic user record creation
     • Automatic profile record creation
     • UUID primary keys
     • Proper foreign keys
     • Uniqueness constraints on email
     • Timestamps on all records

🚀 QUICK START (5 Minutes)

  1. Install Dependencies
     $ cd "c:\wamp64\www\Therapy Booking\backend"
     $ composer install

  2. Configure Environment
     $ cp .env.example .env
     # Edit .env with your database credentials

  3. Import Database Schema
     $ mysql -u root therapy_booking < "..\schema.sql"

  4. Start Server
     $ php -S localhost:8000 index.php

  5. Test Registration
     $ curl -X POST http://localhost:8000/api/auth/register \
       -H "Content-Type: application/json" \
       -d '{"email":"test@test.com","password":"Pass123","userType":"patient","fullName":"Test User"}'

     ✓ Should return 201 with access_token
